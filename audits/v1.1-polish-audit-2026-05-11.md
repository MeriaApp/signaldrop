# SignalDrop v1.1 — adversarial polish audit

**Date:** 2026-05-11
**Author:** Claude Opus 4.7 + Jesse Meria
**Lens:** Would Apple ship this on their main product? Best-in-the-world standard, not "competent for our size."
**Status:** All v1.1 features built (scanner, signal graph, history, per-network, PDF, settings, min-duration, quiet hours). v1.0.2 is WAITING_FOR_REVIEW in ASC. Plan: ship v1.1 as consolidated bump after polish round; cancel + replace 1.0.2.

---

## How the audit was done

1. Read source of every user-facing surface (`MenuBarController.swift`, `NetworkInsightsView.swift`, `ConnectionHistoryView.swift`, `SettingsView.swift`, `OnboardingView.swift`, `ConnectionHistoryPDF.swift`).
2. Captured live screenshots: menubar icon, dropdown menu, Scanner tab, Signal Graph tab, Connection History tab (24h with real seeded data showing C grade), Settings window (defaults + quiet-hours-enabled state), PDF export output.
3. Compared every surface against the four Mac App Store competitors: WiFi Signal ($4.99), Wifiry ($9.99), WiFi Explorer ($19.99), NetSpot (free + IAP).
4. Adversarial framing: "find 5 ways this will break / disappoint / underwhelm."

## The 24 findings, by severity

### TIER 1 — Showstoppers ("this app feels dead" tier)

#### 1.1 Menu bar icon is static
The status item uses `dot.radiowaves.left.and.right` always, regardless of state. User can't glance up and tell if WiFi is connected, weak, or down. Apple's own WiFi menu icon shifts with strength. WiFi Signal does this. We don't.

**Fix:** State-driven SF Symbol. Map:
- `wifi` full (4-bar look) when RSSI ≥ -50
- `wifi` reduced intensity when -65 ≤ RSSI < -50
- `wifi.exclamationmark` when WiFi connected but `NetworkMonitor.isInternetReachable == false`
- `wifi.slash` (red tint) when disconnected or WiFi off
- Update in `MenuBarController.renderStatus()` after every state change

Optional: add the connected SSID as a short text label after the icon for power users (`button.title = state.ssid`). Settings toggle for "Show network name in menu bar".

---

#### 1.2 Signal Graph Y-axis is fixed at -100..-20 dBm
When the user's signal sits at -50 (typical), the bottom half of the chart is wasted empty space. The line hugs the top.

**Fix:** Auto-scale to actual data range with sensible padding.
- Compute `minRSSI` and `minNoise` from `model.samples`
- Domain: `(min - 10) ... max(-30, maxRSSI + 5)` (don't show empty space below -100 if data only goes to -85; don't crowd the top)
- Round to nearest 10 for clean tick marks
- Fall back to `-100...-20` if zero samples

File: `Sources/SignalDrop/NetworkInsightsView.swift`, `SignalGraphPane`.

---

#### 1.3 Scanner shows raw BSSID hex with no vendor lookup
`8c:e9:b6:13:77:e7` is a hex MAC. WiFi Explorer turns this into "Cisco Meraki" via IEEE OUI database. Every other scanner does this.

**Fix:** Bundle a static OUI lookup table (first 3 bytes → vendor name). Sources:
- Free: `https://standards-oui.ieee.org/oui/oui.txt` (~26k entries, large)
- Pragmatic: bundle a curated subset (~500 entries) covering the top vendors (Apple, Cisco, Ubiquiti, TP-Link, Netgear, Aruba, Eero, Asus, Linksys, etc.) — 95% hit rate for ~10KB instead of 4MB
- Or pull from `https://api.macvendors.com/{mac}` on-demand with caching

Implementation:
- `Resources/oui-vendors.json` — bundle ~500-1000 top entries
- `OUILookup.swift` — `func vendor(for bssid: String) -> String?`
- Add vendor as subtitle row in scanner Network column: SSID over BSSID + vendor

---

#### 1.4 Dropdown menu has 20+ items + duplicate toggles
After adding the Settings window, the menu still has Sound Alerts and Signal Warnings toggles at the bottom — they duplicate Settings AND can disagree (menu reads `UserDefaults.standard.bool(forKey: "soundEnabled")`, Settings reads `@AppStorage("soundEnabled")` — same underlying key but separate state observation paths). "Show Welcome…" looks like dev-debug exposure.

**Fix:**
- Remove `soundToggle`, `signalWarningToggle` from menu (Settings is the source of truth — link to it instead)
- Remove "Show Welcome…" item (or move behind `⌘⌥W` keybind without menu visibility — keep callable but un-discovered)
- Consolidate the three export options: "Export Log (CSV)", "Generate ISP Report…", "Copy Receipt for Support" → keep CSV + Copy Receipt; remove "Generate ISP Report" (the new History tab → Export PDF replaces it functionally).
- Audit final item count: target 8-10 items max

File: `Sources/SignalDrop/MenuBarController.swift` `setup()`.

---

#### 1.5 No hover tooltip on Signal Graph
The chart shows lines but no way to read exact RSSI at a specific moment. Apple's Stocks app, every Apple Health chart, every NetSpot graph — all have this.

**Fix:** Swift Charts has `.chartOverlay { proxy in ... }` for hit-testing. Track mouse position, find nearest sample, show floating annotation:
```
9:17:43 PM
RSSI: -52 dBm
Noise: -95 dBm
SNR: 43 dB
```
Vertical guide line at the time cursor. Annotation float aligned to data point.

File: `Sources/SignalDrop/NetworkInsightsView.swift`, `SignalGraphPane`.

### TIER 2 — High-value polish

#### 2.6 TX rate fake-scaled onto dBm axis
Lines 274–284 of `NetworkInsightsView.swift`: `y: .value("dBm-scaled", -100 + Int(s.transmitRate / 20))`. Maps 0–2000 Mbps onto the dBm axis. Technically wrong + visually deceptive when overlaid with real RSSI lines.

**Fix:** Two options:
- (A) Dual Y-axis: dBm on left, Mbps on right via `.chartYAxis` with two `AxisMarks` blocks at different positions
- (B) Separate TX rate into its own row below the main chart (50pt tall)

Option B is simpler and aligns better with Apple's Stocks pattern (price chart on top, volume on bottom). Recommend B.

---

#### 2.7 Outage rows aren't clickable
Connection History tab → Outages section → Table. Click does nothing. User can't see what was happening when the drop hit.

**Fix:** Selection state + detail sheet:
- `@State private var selectedOutage: OutageRecord?`
- `.onTapGesture` or Table's selection binding
- Sheet shows: signal context (RSSI at disconnect + 30s prior), internet status, suggested cause, "Copy details" button, "Show in Signal Graph" button (jumps to graph tab with time range pre-scrolled)

File: `Sources/SignalDrop/ConnectionHistoryView.swift`, `HistoryOutagesSection`.

---

#### 2.8 Connected network not visibly highlighted in scanner
Only a text color change (`.accentColor`) on the SSID. Easy to miss when scanning 20+ rows.

**Fix:** Leading badge ("● Connected" in accent color) + subtle accent-tinted row background. File: `NetworkInsightsView.swift`, `NearbyNetworksList`.

---

#### 2.9 Signal-degraded warnings have no minimum-duration
The phantom-drop suppression added in this session only applies to `.disconnected` events. A 1-second flicker to -76 dBm fires "Signal Weak" notification.

**Fix:** Add `minSignalDegradedDurationSeconds` to `NotificationSettings` (default 10s). In `WiFiMonitor.linkQualityDidChangeForWiFiInterface`, debounce the signal-degraded event: don't emit until the threshold has been below `signalWarningThreshold` continuously for the duration. Same pattern as the disconnect timer.

Files: `Sources/SignalDrop/NotificationSettings.swift`, `WiFiMonitor.swift`, `SettingsView.swift`.

---

#### 2.10 No "test notification" button in Settings
User configures per-event toggles + quiet hours + sound, but can't preview. Tedious to verify by waiting for a real disconnect.

**Fix:** Below the Sound section, a "Send test notification" button that fires the disconnect notification path with a `Test — this is what a disconnect notification looks like` body. Disabled when notifications are denied at the OS level.

---

#### 2.11 Empty states are plain
Scanner's "Scanning nearby networks…" — single gray wifi icon + text. Outages' "No disconnects in this period." — checkmark seal + text. History "no data" looks fine but Scanner could shine.

**Fix:** Animated `wifi` SF Symbol via `.symbolEffect(.variableColor.iterative)` for the scanner empty state. Subtle, doesn't distract. Available macOS 14+.

---

#### 2.12 Grade calculation is opaque
User sees "C — Fair, occasional issues" but doesn't know the math or how to improve.

**Fix:** Hover tooltip on the grade pill in `HistorySummaryCard`:
```
Grade calculation:
  Base score: 100
  4 disconnects × -8 = -32
  1m 45s downtime × -3/min = -5
  Final: 63 → C (75 is B threshold)
```
Add `.help(...)` modifier to the Text. Optionally make the grade clickable to open a "How is this scored?" sheet.

### TIER 3 — Final smoothing

#### 3.13 "Likely cause" column shows "—" for all old events
Old events (pre-disconnect-classifier) have no `details` field. Column shows "—" everywhere for users with old data.

**Fix:** Suppress the column entirely when 100% of visible rows have `cause == nil`, or replace blanks with a contextual note ("Older event — cause not recorded").

---

#### 3.14 No SignalDrop wordmark / logo in PDF
Text-only "SignalDrop" header. An icon next to it would feel branded.

**Fix:** Bundle a small (~24pt) PNG of the app icon in `Resources/`. Inline in `ConnectionHistoryPDFView.header`.

---

#### 3.15 PDF filename uses em-dash
`SignalDrop Receipt — 24h — 2026-05-11.pdf` — em-dashes (` — `) can break on Windows in zipped attachments or some email clients.

**Fix:** Replace with hyphen-minus: `SignalDrop Receipt - 24h - 2026-05-11.pdf`. File: `NetworkInsightsController.defaultPDFFilename`.

---

#### 3.16 Quiet hours has hour granularity only
"10 PM" not "10:30 PM" — annoying for people whose work day ends at 6:30.

**Fix:** Replace the `Picker` with `DatePicker(.hourAndMinute, .compact)`. Store as `Date` components (hour + minute). Update `isInQuietHours` comparison.

---

#### 3.17 No sort/filter in Scanner
Always signal-desc. No way to sort by name, channel, security.

**Fix:** `Table` natively supports column sorting via `Table(model.networks, sortOrder: $sortOrder)`. Define `KeyPathComparator`s for each column. Minor wiring.

---

#### 3.18 No band filter
At 50+ networks, user can't show only 5 GHz.

**Fix:** Add a `Picker` to the toolbar (next to Rescan): `All / 2.4 GHz / 5 GHz / 6 GHz`. Filter `model.networks` accordingly.

---

#### 3.19 No pause/freeze on Signal Graph
Chart auto-scrolls. User wanting to inspect a moment 10s ago gets pushed off.

**Fix:** Toolbar button "Pause" (`pause.fill` / `play.fill` SF Symbol). Toggle stops the sampleStore but keeps the buffer visible.

---

#### 3.20 No time-range presets on Signal Graph
Always shows last 5 minutes.

**Fix:** Toolbar segmented control: `1m / 5m / 15m / 1h`. Wire to `SignalSampleStore.capacity`. 1h = 3600 samples at 1Hz (manageable). 24h would be 86k — not for v1.

---

#### 3.21 No "last updated" indicator
On scanner: when was the last scan? On graph: nothing. On history: refresh button next to PDF, but unclear what.

**Fix:** Subtle bottom-right footer in each tab: `"Last scan 3s ago"` / `"Updated just now"` / `"Auto-refreshing"`. Updates via a 1s timer when in foreground.

---

#### 3.22 Onboarding is 4 sheets
Modern macOS apps use one progressive screen with permissions surfaced inline.

**Fix:** Larger refactor — not in this round. Defer to v1.2 or after v1.1 ships.

---

#### 3.23 Accessibility unaudited
VoiceOver labels on custom tab bar, custom toggle buttons in Signal Graph, segmented control likely missing. Color-coded grade pill needs ensure text fallback is sufficient.

**Fix:** Audit pass with `.accessibilityLabel` modifiers added to every custom-button/custom-control. Run VoiceOver and walk through every screen. Half-day of work — schedule separately.

---

#### 3.24 "Show Welcome…" exposes dev path
End users never need to re-run onboarding manually.

**Fix:** Remove from menu (covered in 1.4). Keep the OnboardingController code path callable internally (for testing or first-launch detection) but un-surfaced in production UI.

## Known dev-environment gotchas (next session must know)

1. **Two event databases exist.** The original (non-sandboxed) build wrote to `~/Library/Application Support/SignalDrop/events.db`. The current sandboxed Debug build writes to `~/Library/Containers/com.meria.signaldrop/Data/Library/Application Support/SignalDrop/events.db` — empty until events accumulate. To test History tab with real data, copy the older DB into the container path (it's a 40-event historical set from non-sandboxed builds). Memory: `reference_signaldrop_dual_event_db_sandbox_gotcha.md`.

2. **SwiftUI custom controls aren't accessible via osascript / System Events.** The Picker, custom tab buttons, custom toggle buttons all fail accessibility hierarchy queries. To trigger them in automation: use keyboard shortcuts (`⌘1/⌘2/⌘3` for tabs, `⌘E` for Export PDF, `⌘,` for Settings) or cliclick at known coordinates. AppleScript `click button "Export PDF…"` will FAIL.

3. **xcodegen generate** must be run after adding new source files to `Sources/SignalDrop/`. The .xcodeproj is generated, not hand-maintained.

4. **APPSTORE compile flag is set on Debug builds too** (see `project.yml`). This means the Direct-only `#if !APPSTORE` paths (UpdaterService, WebhookService, Disconnect-from-current-network) are unavailable in Debug. Use `SignalDropDirect` scheme to test those.

5. **SourceKit's in-IDE diagnostics lag the actual compiler.** "Cannot find type X" errors in the diagnostic panel are usually stale — if `xcodebuild` succeeds with no `error:` lines, the code is fine. Ignore the squigglies.
